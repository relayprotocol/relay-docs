Relay supports a variety of testnet and tokens. Using testnets and their tokens is a great way to test bridging with no cost. Relay's testnet chains are fully
supported across all of Relay's tools: APIs, SDK, and UI Kit. Feel free to explore the chains offered below. For the latest list, please explore the
[Testnets Supported Chains](/references/api/api_resources/supported-chains#testnets).

<Tip>**Testnets are not recommended for testing swaps.** Testnets typically lack real DEX liquidity making testnet swaps fail. Consider using
inexpensive L2s like Base or Arbitrum to test swapping. The cost of gas will outweigh the time required to debug testnet-specific issues like illiquidity.</Tip>

## API Support

To access testnets with the Relay API, make sure to use the correct base URL: `https://api.testnets.relay.link` By using this base URL, you'll be able
to bridge, swap, and transact on testnets Relay supports. To get started, check out the [API documentation](/references/api/overview).

## SDK Support

To access testnets with the Relay SDK, make sure to use the correct base URL: `https://api.testnets.relay.link` You'll pass this base URL in
the `baseApiUrl` parameter when creating the client. To get started, check out the [SDK documentation](/references/relay-kit/sdk/installation).

## UI Kit Support

To access testnet with the Relay UI kit, make sure to pass the testnet chain you want e.g. `base sepolia`, `sepolia`, etc.
To get started, check out the [UI Kit documentation](/references/relay-kit/overview).
