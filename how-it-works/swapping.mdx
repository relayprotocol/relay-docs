### Introduction

Swapping is the act of taking one token and exchanging it for another. There are many protocols that help with this, notably, UniSwap, PancakeSwap, CowSwap, and many more. These protocols can either center around decentralized liquidity or centralized liquidity, with various different caveats for different use cases.

The act of crosschain swapping usually requires multiple steps by the user, each using a different protocol:

1. Swapping from an ERC20 to a native asset
2. Going to a bridge and bridging the native asset
3. Swapping from a native asset to an ERC20 on the destination

With Relay, these steps are bundled into one operation the user can take for the sake of cost and speed efficiency.

### Crosschain Swaps via Relay

Relay builds on top of its pre-existing bridging infrastructure to enable crosschain swapping. The bridging infrastructure allows the fast transfer of native and some preset ERC20 assets across chains.

This extends to swapping by helping the user swap out of any asset into a native asset on the origin, with the solver as the destination. And with bridging, users can get into any asset they want on the destination, using the solver's native balance on that chain.

Relay abstracts the complicated swapping layer and under the hood uses a combination of protocols to help find the best prices for swapping. This way the user does not need to check multiple swap protocols to find the best price.

### User Experience

For the client, it feels like just a regular swap. Relay bundles multiple origin and crosschain swaps into one simple and fast user operation. Usually the flow is something along:

- If it's an ERC20 input:
  - Approve the solver to spend your token on origin
  - The protocol executes the swap on origin and the bridge, including any extra destination swaps or call execution (like mints)
- If it's an ETH input:
  - Send ether to the solver
  - The protocol executes the bridge, including any extra destination swaps or call execution (like mints)

Relay's infrastructure is built to provide the best quotes, which is why it also supports same-chain swapping. With same-chain swapping, either the user can self-execute the swap, or Relay can coordinate executing the swap in situations where the user does not have enough gas to pay for the transaction.
